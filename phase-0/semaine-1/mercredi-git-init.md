# Mercredi Sem.1 — Git versioning local

## Ce que j'ai fait
- Compris pourquoi versionner dès le début
- Initialisé mon premier repo Git local
- Fait mon premier commit

## Commandes apprises

git init               → initialiser un repo Git dans un dossier
git status             → voir quels fichiers sont modifiés
git add fichier.txt    → préparer un fichier pour le commit
git add .              → préparer TOUS les fichiers modifiés
git commit -m "msg"    → enregistrer une photo du travail
git log                → voir tout l'historique des commits
git log --oneline      → historique en version courte
git diff               → voir ce qui a changé avant de commiter

## Les 3 zones Git — très important à retenir

Dossier de travail → (git add) → Zone d'attente → (git commit) → Repo

Explication :
- Dossier de travail = tes fichiers sur le PC
- Zone d'attente (staging) = fichiers prêts à être sauvegardés
- Repo = historique officiel de toutes les photos

## Pourquoi Git est utile en réseau et sécu
- Garder l'historique de toutes mes configurations
- Revenir en arrière si une config casse tout
- Preuve visible de mon travail pour les recruteurs
